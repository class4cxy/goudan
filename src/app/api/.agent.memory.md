## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

Next.js App Router API 路由目录，所有路由均设置 `export const runtime = "nodejs"`（禁用 Edge Runtime，保证 better-sqlite3 和 Node 原生模块可用）。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
api/
├── chat/route.ts           # AI 对话流式接口（POST，SSE 流式响应）
├── status/route.ts         # 机器人状态聚合接口（GET，代理 Bridge）
├── tasks/route.ts          # 定时任务管理（GET/PATCH/DELETE）
├── voice-logs/
│   ├── route.ts            # 语音记录天列表（GET，最近 90 天）
│   └── sessions/route.ts  # 语音 session 列表或消息详情（GET ?day= 或 ?session_id=）
└── threads/
    ├── route.ts            # 线程列表/创建（GET/POST）
    └── [id]/route.ts       # 单个线程操作（按需查看）
```

**各接口职责：**

**`/api/chat` (POST)**
- 接收 `{messages: UIMessage[], threadId?}` 
- 调用 `streamText` + `ALL_TOOLS`，最多 8 步
- 完成后异步保存消息到 `thread_messages`，自动设置线程标题（取第一条用户消息前 60 字）
- 返回 SSE 流式响应（`toUIMessageStreamResponse()`）

**`/api/status` (GET)**
- 并行请求 Bridge 的 `/status` 和 `/health`
- 超时 5 秒，Platform 不可达时返回 `{platform_ok: false}`
- 被侧边栏 `useRobotStatus` hook 轮询调用

**`/api/tasks` (GET/PATCH/DELETE)**
- GET：列出所有定时任务
- PATCH：`{id, enabled}` 启用/禁用任务（调用 `toggleTask`）
- DELETE：`{id}` 删除任务（调用 `removeTask`）
- 使用 Zod 对请求体做 schema 验证

**`/api/threads` (GET/POST)**
- GET：列出所有线程（按 updated_at 倒序）
- POST：`{id}` 创建新线程（幂等）

**`/api/voice-logs` (GET)**
- 返回最近 90 天中有语音记录的日期列表（UTC+8），含每天 session 数和消息数

**`/api/voice-logs/sessions` (GET)**
- `?day=YYYY-MM-DD`：返回当天所有 session（started_at/ended_at/message_count）
- `?session_id=xxx`：返回该 session 的所有消息（role/content/created_at）

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- `/api/chat` 中 `onFinish` 回调构建 assistant UIMessage 的逻辑较复杂（手动拼接 step-start / text / dynamic-tool parts），与 Vercel AI SDK 版本强耦合，升级 SDK 时需重点检查
- `/api/status` 没有缓存，每次轮询都直接打 Bridge，高频刷新时有资源浪费

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **`export const maxDuration = 60`**（`/api/chat`）：Vercel 部署时允许函数最长运行 60 秒，适配多步 Agent 调用耗时
- **线程标题自动提取**：仅在 `thread.title` 为空时设置，取第一条用户消息文本前 60 字，不会覆盖已有标题

## 注意事项

> 描述隐式约定、相似功能函数的区分与选择指南。

- 所有路由必须加 `export const runtime = "nodejs"`，否则 Edge Runtime 下 `better-sqlite3` 无法运行
- `threadId` 在 `/api/chat` 中是可选的——不传则不保存消息历史（无状态对话）
- Zod 验证失败时统一返回 `{error: "参数错误"}`，HTTP 400
