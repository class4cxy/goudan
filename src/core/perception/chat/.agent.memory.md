## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

文字对话感知通道（对称于 `perception/audio/`）。Web Chat 是与语音并列的感知输入方式，两者进入 Brain 后处理路径完全一致。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
chat/
└── index.ts          # notifyChatInput() / notifyChatComplete() — Spine 侧链事件发布
```

- **`notifyChatInput(text, threadId)`**：在 HTTP 流式推理开始前调用，向 Spine 发布 `sense.chat.message`，让 ConversationManager 等模块感知到用户活动
- **`notifyChatComplete(threadId)`**：推理完成后调用，发布 `sense.chat.complete`，可用于重置空闲计时器

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **双路径设计**：文字对话不经过 Spine → Brain（因为 SSE 流式响应需要同步返回），而是直连 Brain；Spine 侧链事件只用于通知其他模块（状态同步、空闲计时器重置等）
- **不做 STT 转换**：文字本身就是语义信号，无需像音频那样经过 Whisper 转写，直接进入 Brain 的 messages 参数

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- `sense.chat.message` 和 `sense.chat.complete` 事件类型已在 `runtime/spine/types.ts` 的 `SpineEventType` 中注册
- 扩展时，其他文字输入通道（微信消息、家庭内网 API 等）也应在此目录下添加对应适配器，发布相同的 Spine 事件类型
