## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

Agent 神经系统核心目录（替代原 `src/lib/`）。包含所有具有 Agent 业务语义的模块，按神经系统职能划分为四层。`src/lib/` 保留用于零业务属性的通用基础设施（db、utils）。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
core/
├── runtime/                ← 物理基底（神经介质，无业务逻辑）
│   ├── spine/              ←   事件总线 + L0 工作记忆（核心）
│   └── platform-connector/ ←   Platform 硬件事件 ↔ Spine 事件双向适配
│
├── perception/             ← 多通道感知层（原始信号 → 语义事件）
│   ├── audio/              ←   语音：VAD → STT → 情绪/唤醒词 → Spine（含 TTS 效应器）
│   └── chat/               ←   文字：Web Chat → Brain 直连 + Spine 侧链事件
│
├── cognition/              ← 认知层（思维 · 记忆 · 能力）
│   ├── brain/              ←   LLM 推理 + Prompt（语音对话 / 定时任务）
│   ├── memory/             ←   分级对话记忆 L1-L4（ConversationBuffer）
│   └── tools/              ←   Agent 工具注册表（ALL_TOOLS，14 个工具）
│
└── behavior/               ← 行为层（认知 → 输出模式）
    ├── conversation/       ←   语言行为：状态机 + 主/被动发起器
    ├── motor/              ←   运动行为：导航意图 + 电机指令转发
    └── scheduler/          ←   自主定时任务（自主神经系统）
```

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- `perception/` 未来扩展：`vision/`（环境持续感知）、`spatial/`（SLAM 语义解读）

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **四层架构命名**（用户指定）：runtime / perception / cognition / behavior，对应神经系统的物理基底 / 感知处理 / 认知推理 / 行为输出
- **`lib/` 与 `core/` 的分界**：`lib/` = 零业务属性（db、utils）；`core/` = 有 Agent 语义的所有模块
- **`tools/` 替代 `agent/`**：整个系统才是 agent，原 `agent/` 目录只是工具注册表，命名歧义
- **`tools/` 扁平化**：原 `agent/tools/` 子目录合并到 `tools/` 根目录，减少嵌套层级

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- 所有层间通信走 Spine（`runtime/spine`），模块之间不直接调用
- Brain 只发布事件，不直接调用效应器
- 模块启动顺序（在 `instrumentation.ts` 中）：PlatformConnector → AudioThalamus → AudioEffector → ConversationManager → 主动发起器
- `perception/audio/effector.ts` 中的 `PlatformConnector.send()` 是唯一允许直接调用 PlatformConnector 而非通过 Spine 的场景（TTS 是同步音频输出，必须绕过事件队列）
