## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

Platform ↔ Spine 双向适配器。作为 WebSocket 客户端连接 Platform（Python FastAPI `/ws`），将硬件事件翻译为 SpineEvent 发布到 Spine，同时将 Spine 的 action 事件转发回 Platform 执行。是两个运行环境（Python/Node.js）之间唯一的通信桥梁。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
platform-connector/
└── index.ts    # PlatformConnector 单例，WebSocket 客户端 + 双向事件翻译
```

**核心 API：**
- `PlatformConnector.start()` — 启动 WebSocket 连接 + 订阅 Spine outbound 事件
- `PlatformConnector.send(message)` — 向 Platform 发送指令（供 AudioEffector 等调用）

## INBOUND_MAP 当前映射

| Bridge 事件 | Spine 事件 | 优先级 | 说明 |
|------------|-----------|--------|------|
| `sense.audio.speak_end` | `sense.audio.speak_end` | LOW | TTS 蓝牙播放完毕，供 ConversationManager 状态同步 |
| `sense.power.low_battery` | `sense.system.battery` | HIGH | 低电量告警 |

**已移除的映射（Chat 模式变更）：**
- `sense.audio.speech_start` — 录音由手机端负责，Platform 不再采集
- `sense.audio.speech_end` — 同上

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- `INBOUND_MAP` 映射较少，摄像头/电机等感知事件类型需后续补充
- 断线重连是简单定时重连，无指数退避，频繁断线时会有轻微风暴
- `sense.lidar.scan` 不在 `INBOUND_MAP` 中（Node.js 侧不消费点云，只有 Python SLAM 和前端可视化使用），平均 10Hz 广播；若需抑制该流量可在 Bridge 侧设 `LIDAR_BROADCAST_EVERY` 降频

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **Node.js 作为 WS 客户端，Platform 作为 WS 服务端**：Platform 已是 FastAPI 服务（端口 8001），在其上加 `/ws` 端点自然，Node.js 连入符合已有的调用方向
- **断线消息暂存并重放**：`send()` 在断线时不再直接丢弃 action 消息，改为入队（上限 50 条）并在重连后补发，降低 `action.speak` 丢包导致的 `speak_end` 缺失

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- 连接地址通过环境变量 `PLATFORM_WS_URL` 配置，默认 `ws://localhost:8001/ws`
- 新增硬件事件类型时，需在 `INBOUND_MAP` 中添加对应映射（type + priority + summary）
- 新增需转发给 Platform 的 action 类型时，在 `OUTBOUND_TYPES` 数组中追加即可
