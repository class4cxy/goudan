## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

`src/lib/spine/` 是整个 Home Agent 的**神经传输中枢**，对应人体脊柱（Spine）的角色。所有感知事件（来自 Sensor/Thalamus 层）和行动指令（来自 Brain 层）都通过 Spine 传输，模块之间不直接通信，只和 Spine 交互。

架构分层：`Sensor → Thalamus → [Spine] → Brain → [Spine] → Dispatcher → Effector`

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
spine/
├── types.ts     # 所有事件类型定义：SpineEventType、Payload 接口、MemoryEntry、优先级
└── index.ts     # SpineClass 实现 + 全局单例导出（globalThis.__spine）
```

**核心 API：**

- `Spine.publish(event)` — 发布事件，必须提供 `summary`（人类可读摘要，进入工作记忆）
- `Spine.subscribe(types[], handler)` — 订阅事件，返回 `Unsubscribe` 函数
- `Spine.getWorkingMemory(windowMs?)` — 获取近期 MemoryEntry 列表（默认 10 分钟窗口）
- `Spine.formatMemoryForLLM(windowMs?)` — 将工作记忆格式化为 LLM 可直接读取的文本块

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- 当前工作记忆仅保存 `MemoryEntry`（摘要），不保存原始 `payload`，LLM 无法回溯完整事件数据；如需审计或回放，未来需要独立的事件持久化层（写入 SQLite）
- `setImmediate` 在浏览器环境不可用；Spine 仅设计用于 Node.js 服务端，不可在 React 客户端组件中引用

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **命名为 Spine 而非 Channel/Kernel**：用户明确指出该组件职责是"传输"，`Channel` 语义过于被动，`Kernel` 语义为内核/计算，`Spine`（脊柱）在神经系统隐喻体系中精准对应"信号传输主干"。见 [架构讨论](../../../docs/)
- **publish 强制要求 summary 字段**：所有事件必须附带人类可读摘要，确保工作记忆天然对 LLM 友好，不需要后处理
- **CRITICAL 事件跳过 setImmediate**：优先级为 CRITICAL 的事件直接用 `Promise.resolve()` 调度，减少一个 event loop tick 的延迟

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- 事件类型命名规范：`<层级>.<来源>.<信号>`，如 `sense.audio.transcript`、`action.speak`
- 新增事件类型时，**必须同时**在 `types.ts` 中添加对应 Payload 接口，保持类型安全
- `getWorkingMemory()` 返回的是只读摘要列表；`formatMemoryForLLM()` 是其格式化版本，两者底层数据相同，按需选择
- 单例通过 `globalThis.__spine` 保持，Next.js 热重载不会重置，订阅关系在开发时会累积——重启服务可清空
