## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

`src/lib/brain/` 是 Home Agent 的**统一大脑推理层**，对应人体的大脑皮层。所有 LLM 推理调用集中于此，避免各模块各自持有模型实例，保证模型配置的单一来源。

与周边模块的分工：
- `audio/`（丘脑）：负责感知转换，把原始信号变成语义事件
- `brain/`（大脑）：负责推理决策，知道"说什么"和"做什么"
- `conversation/`（语言中枢）：负责交流调度，管理"什么时候说"
- `agent/tools/`（工具库）：负责执行能力，是大脑的"手脚"

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
brain/
├── index.ts         # 模型定义（AGENT_MODEL）+ 统一 re-export 入口
├── prompts.ts       # 固定行为规则 + buildSystemPrompt（注入 Bootstrap + 时间 + 历史摘要）
├── bootstrap.ts     # BootstrapLoader：加载根目录 ARIA.md / HOME.md / FAMILY.md
├── conversation.ts  # 语音对话推理：generateVoiceResponse()，流式分句输出
└── task.ts          # 定时任务推理：executeScheduledTask(task, tools)
```

**数据流：**
```
brain/conversation.ts ← ConversationManager（manager.ts）调用
brain/task.ts         ← agent/index.ts 注入 ALL_TOOLS 后导出 executeScheduledTask
brain/prompts.ts      ← 所有需要 system prompt 的地方（conversation/task/chat API）
brain/bootstrap.ts    ← 被 prompts.ts 单例引用，进程启动时读一次文件并缓存
```

**Bootstrap 文件（根目录，由 BootstrapLoader 加载）：**

| 文件 | 内容 | 支持占位符 |
|---|---|---|
| `ARIA.md` | 人格与知识面描述（首行可用 `{AGENT_DISPLAY_NAME}`，见 `bootstrap.ts`） | 是（`AGENT_DISPLAY_NAME`） |
| `HOME.md` | 家中设备能力与房间布局 | 否 |
| `FAMILY.md` | 家庭成员模板（`{ARIA_FAMILY_*}` → env） | 是 |

**buildSystemPrompt 组成顺序：**
```
Bootstrap（ARIA.md + HOME.md + FAMILY.md）  ← 可变，修改文件后重启生效
  ↓
buildBehaviorRules（固定行为准则/电量规则） ← 与代码逻辑强耦合，改 prompts.ts
  ↓
当前时间                                   ← 每次请求动态注入
  ↓
历史摘要（ConversationBuffer）             ← 可选，由 chat route 注入
```

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- `brain/task.ts` 的 tools 参数类型为 `Record<string, unknown>` 做了类型宽松，待 AI SDK 提供更好的工具类型后收紧
- 当前所有推理统一使用 deepseek-chat，未来可考虑按场景选择不同模型（语音对话用更快的模型，任务执行用更强的模型）
- `generateVoiceResponse` 中主动发言时注入的占位消息 `（主动发言）` 是 workaround，根本原因是 AI SDK `streamText` 强制要求 messages 非空；待 SDK 放开限制后可移除

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **brain/ 独立于 agent/**：用户明确指出 `conversation/brain.ts` 不符合架构设计，Brain 应该是独立的统一目录，不寄生在任何功能模块下
- **tools 通过参数注入到 brain/task**：`brain/task.ts` 不直接 import `agent/index.ts`，避免 brain→agent→brain 的循环依赖；由 `agent/index.ts` 组装后导出 `executeScheduledTask`
- **agent/prompts.ts 保留为 re-export**：现有 `app/api/chat/route.ts` 等从 `@/lib/agent/prompts` 导入，保留 re-export 壳避免修改大量文件
- **Bootstrap 文件体系**：Aria 人格、家中设备、家庭成员等可变内容从代码迁移到根目录 Markdown 文件（`ARIA.md` / `HOME.md` / `FAMILY.md`），修改文件后 PM2 重启即生效，无需改代码。`FAMILY.md` 中 `{ARIA_FAMILY_*}` 占位符在加载时从环境变量替换。
- **SYSTEM_PROMPT 保留为向后兼容 getter**：调用 `buildSystemPrompt()` 生成，供旧代码静态引用；新代码直接调用 `buildSystemPrompt()`
- **语音回复优先用提示词约束纯文本输出**：语音场景不做后处理格式清洗，依赖 `VOICE_ADDENDUM` 强约束模型输出自然段纯文本，避免策略分叉（来源：用户语音场景反馈）
- **语音回复不设固定句数上限**：用户要求保留讲故事/长解释能力，因此 `VOICE_ADDENDUM` 改为“按需求自适应长度”，并允许礼貌、关怀、玩笑、撒娇等缓和气氛语气；仅保留“纯文本禁 Markdown”硬约束（来源：用户语音场景反馈）

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- 新增推理场景 → 在 `brain/` 下新建文件，不要在功能模块内直接 `new OpenAI()` 或 `streamText()`
- `AGENT_MODEL` 从 `brain/index.ts` 导出；`@/lib/agent` 的 re-export 仅为向后兼容，新代码直接 `import from '@/lib/brain'`
- **修改助手显示名** → `.env` 中 `AGENT_DISPLAY_NAME`（默认「狗蛋」），与同模块 `src/lib/agent-display.ts` 一致；重启生效
- **修改人格长文 / 家中设备 / 家庭成员** → 编辑根目录对应 `.md` 文件，重启进程生效；`ARIA.md` 可与 env 占位符组合
- **修改行为规则/工具调用规则/电量阈值** → 编辑 `prompts.ts` 的 `buildBehaviorRules()`（与代码逻辑耦合，不宜放 Markdown）
- `bootstrapLoader.invalidate()` 可手动清除缓存（如实现文件监听热更新），正常流程无需调用
- `generateVoiceResponse` 的 `triggerNote` 参数：主动发起时传入情景描述，让 LLM 知道为什么主动说话；被动响应时不传
- **messages 不能为空**：AI SDK `streamText` 要求 messages 数组非空。`generateVoiceResponse` 内部处理了此边界：当 context 无历史（主动发言冷启动）时自动注入占位消息，调用方无需额外处理
- `VOICE_ADDENDUM` 采用纯文本规则描述（不使用 Markdown 列表样式），并加入“输出前自检重写”约束，降低模型在语音场景复用 Markdown 风格的概率
