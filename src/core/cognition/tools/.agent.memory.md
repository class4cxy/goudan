## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

AI Agent 核心层，负责模型配置、工具注册、Prompt 管理和定时任务执行。是 Node.js 侧所有 AI 能力的集中入口，由 `/api/chat` 路由和定时任务调度器共同使用。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
agent/
├── index.ts        # 核心入口：模型实例、ALL_TOOLS 注册表、executeScheduledTask
├── prompts.ts      # System Prompt 定义（buildSystemPrompt / SCHEDULER_PROMPT）
└── tools/          # 各设备工具实现（见 tools/.agent.memory.md）
    ├── power.ts    # getPowerStatus — 查询 INA219 电源状态
    └── ...
```

**关键导出：**

- `AGENT_MODEL`：DeepSeek `deepseek-chat` 模型实例，使用环境变量 `DEEPSEEK_API_KEY`
- `ALL_TOOLS`：所有可用工具的注册表 dict，传给 `streamText` / `generateText`
- `executeScheduledTask(task)`：定时任务执行入口，内部调用 `generateText` + `ALL_TOOLS`，最多 6 步

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- Bridge 已有 `/clean/stop`（原地停止）和 `/clean/zone`（区域清扫）接口，但 `ALL_TOOLS` 中尚未对应封装工具
- 对话模式（`/api/chat`）使用 `streamText`，最多 **8 步**；定时任务模式（`executeScheduledTask`）使用 `generateText`，最多 **6 步**——两处步数上限不同，需注意

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **主模型选用 DeepSeek**（非 OpenAI）：`deepseek-chat`，成本低；视觉分析单独使用 Qwen-VL，两个模型分工明确
- **定时任务执行与对话共用同一套工具**：`executeScheduledTask` 直接复用 `ALL_TOOLS`，Scheduler Prompt 通过 `SCHEDULER_PROMPT` 单独定制，无需维护两套工具

## 注意事项

> 描述隐式约定、相似功能函数的区分与选择指南。

- 新增工具必须在 `ALL_TOOLS` 中注册，否则 Agent 无法调用
- `AGENT_MODEL` 只在此文件定义，对话路由和 Scheduler 都从这里 import，不要各自单独实例化
- `buildSystemPrompt()` 用于对话场景（含当前时间等动态信息），`SCHEDULER_PROMPT` 用于自动执行场景（更精简）
