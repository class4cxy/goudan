## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

运动模块（Node.js 侧）。负责将 Agent 的高层导航意图（`action.navigate`）路由到底层电机执行（`action.motor` → Bridge → Chassis）。当前为激光雷达安装前的存根阶段：导航意图被记录到 Spine 工作记忆，实体执行待激光雷达 + NavigationThalamus 接入后生效。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
motor/
├── effector.ts   # MotorEffector：订阅 action.navigate（存根）+ action.motor → BridgeConnector
└── index.ts      # 模块入口，startMotorModule() 统一启动
```

**数据流（当前阶段）：**
```
Agent navigateTo 工具 → Spine [action.navigate] → MotorEffector（存根，仅记录意图）

内部/测试 → Spine [action.motor] → MotorEffector → BridgeConnector → Bridge chassis
```

**数据流（激光雷达安装后）：**
```
Agent navigateTo 工具
  → Spine [action.navigate]
  → NavigationThalamus（新增）
      → 地图查询路径 → 发布 action.motor 序列
  → MotorEffector → BridgeConnector → Bridge chassis
  ↑
sense.system.obstacle（障碍物检测）→ NavigationThalamus → 动态避障重规划
```

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **Agent 接口设计为目标导向（navigateTo），不暴露时长参数**：用户决策，理由是最终形态为激光雷达地图导航，目标地点不需要时长，时序由 NavigationThalamus 内部控制
- **action.motor 不进 BridgeConnector.OUTBOUND_TYPES**：MotorEffector 手动调用 `BridgeConnector.send()` 以保持与 AudioEffector 一致的模式，避免重复转发
- **Explorer 增加 `EXPLORER_HIGH_LIDAR` 保守模式**：用户反馈“雷达明显高于障碍物却仍前进碰撞”。原因是 LD06 为 2D 平面雷达，车体低位障碍会落在扫描平面以下形成盲区。该模式通过降速/缩短前进脉冲/提高前向谨慎阈值降低风险，但不能替代保险杠或低位测距硬件
- **Explorer 增加“无位移强制脱困”保护**：用户反馈“原地往前走，走不动还走”。前进脉冲后读取 `/slam/pose`，若连续多次位移低于阈值则判定顶住，立即停止前进并执行后退+大角度转向，避免持续顶障碍
- **Explorer 接入超声波前向安全门**：每轮读取 `/ultrasonic/status` 的最新距离；当距离低于 `EXPLORER_ULTRASONIC_STOP_CM` 时直接进入紧急避障，低于 `EXPLORER_ULTRASONIC_TURN_CM` 时优先转向。超声波判定优先级高于雷达“可前进”判定，用于补齐高装雷达低位盲区
- **Explorer 增加小空间防打转策略**：连续原地转向达到阈值（`EXPLORER_MAX_INPLACE_TURNS`）且前方未到急停距离时，执行一次短前探（`EXPLORER_INPLACE_POKE_*`），用于打破“方向正确但一直转不出来”的循环
- **Explorer 增加转向迟滞与强脱困**：当左右侧差距接近时，优先沿用上次转向方向（`EXPLORER_TURN_HYSTERESIS_MM`）减少来回抖动；若连续原地转向仍未脱困，则执行“后退 + 大角度换向”强制跳出，抑制 SLAM 在原地发散
- **Explorer 增加斜角通行增强**：在转向阶段除正前阈值外，增加斜向（30°~50°）出口判据（`EXPLORER_TURN_EXIT_DIAG_MM`）；当识别到明显斜向通道时执行短促引导前探（`EXPLORER_DIAG_GUIDE_POKE_*`），提高斜角门口/斜走廊通过率
- **Explorer 默认禁用后退（用户决策）**：用户反馈“后退经常误判且策略应以持续前进为主，原地转向 360° 足够脱困”。当前默认 `EXPLORER_ALLOW_REVERSE=0`，避障优先“停 + 转 + 前进”；仅在显式设置 `EXPLORER_ALLOW_REVERSE=1` 时启用后退序列
- **前进与运动速度调慢（用户决策）**：用户反馈"前进、运动速度太快"。Explorer 默认前进 30→20、转向 30→20、脉冲 0.4s→0.5s；底盘默认速度 60→45。可通过 `EXPLORER_FORWARD_SPEED`、`EXPLORER_BURST_DURATION`、`EXPLORER_TURN_SPEED`、`CHASSIS_DEFAULT_SPEED` 微调
- **语音/手动与建图速度分离（用户决策）**：用户反馈"前进后退还是快，建图时又异常慢"。两套速度完全独立：语音/手动用 `MANUAL_DEFAULT_SPEED`（Node 侧，默认 35）/`CHASSIS_DEFAULT_SPEED`（Platform  fallback）；建图用 `EXPLORER_FORWARD_SPEED`（默认 28）、`EXPLORER_TURN_SPEED`。互不干扰
- **转向角度过大（用户决策）**：用户反馈"左转、右转、掉头转动角度过大"。语音控制：左转/右转默认 0.85s（约 85°），掉头默认 1.75s（约 175°），不再不设 duration 持续转动。增加「掉头」「调头」映射。**注意**：语音转向与 Explorer 建图转向独立，后者用 `EXPLORER_TURN_STEP_DURATION`、`EXPLORER_ESCAPE_ROTATE_DURATION`，建图需足够转角才能有效脱困
- **转向角度微调（用户决策）**：用户反馈"左转右转角度偏小（~45°）、掉头角度偏大"。算法为开环时长控制（无角度反馈）。左转/右转 0.45→0.85s（约 85°），掉头 2.2→1.75s（约 175°）。支持 `TURN_DEFAULT_DURATION`、`TURN_AROUND_DURATION` 环境变量
- **建图转向不足（误调修复）**：之前为修语音"转动角度过大"误将 Explorer 的 `TURN_STEP_DURATION` 0.4→0.3、`ESCAPE_ROTATE_DURATION` 1.8→1.2，导致建图时"基本不怎么转"。已恢复：0.3→0.45s、1.2→1.6s。语音与建图转向参数完全独立

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- `action.navigate` 是 **Agent facing**（目标级）；`action.motor` 是 **内部实现**（指令级）。新增功能请勿直接让 Agent 发布 `action.motor`
- 激光雷达到来时，`NavigationThalamus` 应放在本目录（`thalamus.ts`），订阅 `action.navigate`，内部发布 `action.motor` 序列
- 真机测试底盘时，直接用 `platform/devices/motor_test.py` 或通过 Platform REST `/motor/command`，无需经过 Spine
- `EXPLORER_HIGH_LIDAR=1` 适用于雷达高装场景；若雷达降低安装高度或加装底盘防撞/测距传感器，应评估是否关闭，以免探索效率过低
- “无进展判定”依赖 SLAM 位姿可用性（`/slam/pose`）；若 SLAM 未启动或漂移严重，该保护可能不触发，可通过 `EXPLORER_PROGRESS_MIN_MOVE_MM` 与 `EXPLORER_STUCK_BURSTS` 调参
- Explorer 启动阶段要求 `/slam/start` 成功；若返回非 2xx（如雷达未连接或 breezyslam 未安装）会直接取消探索，避免“无 SLAM 仍继续前进”的风险
- 超声波读数需要新鲜度校验：超过 `EXPLORER_ULTRASONIC_TTL_MS` 的旧数据不参与决策（避免使用过期距离导致误判）
- 小空间调参优先顺序：先调 `EXPLORER_TURN_STEP_DURATION` / `EXPLORER_MAX_TURN_STEPS` / `EXPLORER_TURN_EXIT_MM`，再调短前探参数；避免直接把前探时长设太大导致碰撞风险上升
- 为减少“走一遍后仍乱走”，默认将 `CLEAR_MM` 下调到更保守值（380mm）并允许通过环境变量微调转向速度/阈值，优先降低原地旋转占比
- 若出现“斜角识别不了”优先调 `EXPLORER_TURN_EXIT_DIAG_MM`（适当下调）与 `EXPLORER_DIAG_GUIDE_POKE_DURATION`（适当上调），并保持超声波急停阈值不变
