## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

SQLite 数据访问层，使用 `better-sqlite3`（同步 API）。数据库文件位于 `data/home-agent.db`，WAL 模式运行。是全项目唯一的持久化存储，Agent 工具、调度器、API 路由均从此处读写数据。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

**数据表（共 7 张）：**

| 表名 | 用途 |
|------|------|
| `conversations` | 历史遗留，暂未使用（已被 threads 体系取代） |
| `threads` | 对话线程元数据（id/title/created_at/updated_at） |
| `thread_messages` | 线程完整消息列表（JSON 序列化，一个 thread 一条记录） |
| `cleaning_records` | 用户触发的清扫记录（rooms/mode/triggered_by/duration/area） |
| `inspection_records` | 摄像头巡检记录（camera_id/score/has_trash/dirty_zones/action_taken） |
| `scheduled_tasks` | 定时任务配置（name/cron/task_type/config/enabled/last_run_at） |
| `preferences` | 键值对系统偏好（目前存 quiet_hours_start/quiet_hours_end，默认 22/7） |

**关键导出：**
- `queries`：所有预编译 SQL 语句的集合对象（`better-sqlite3` prepare）
- `parseRawTask(raw)`：将 `RawScheduledTask`（DB 原始行）转为 `ScheduledTask`（config 字段 JSON 解析）
- `isQuietHour()`：判断当前时间是否在静默时段（读 preferences 表），调度器调用
- 类型：`Thread`、`ScheduledTask`、`RawScheduledTask`

**`queries` 对象主要方法：**

```typescript
// 线程
queries.listThreads.all()           // 列出所有线程（按 updated_at 倒序）
queries.getThread.get(id)           // 获取单个线程
queries.createThread.run(id)        // 创建线程（INSERT OR IGNORE）
queries.saveThreadMessages.run(threadId, jsonStr) // 保存消息列表
queries.touchThread.run(threadId)   // 更新 updated_at
queries.setThreadTitle.run(title, threadId)

// 清扫记录
queries.insertCleaningRecord.run(rooms, mode, triggeredBy, duration, area)
queries.getCleaningHistory.all(limit)

// 巡检记录
queries.insertInspection.run(cameraId, score, hasTrash, dirtyZones, actionTaken)
queries.getInspections.all(limit)

// 定时任务
queries.getAllTasks.all()
queries.insertTask.run(name, cron, taskType, configJson)
queries.updateTaskEnabled.run(enabled, id)
queries.updateTaskLastRun.run(id)
queries.deleteTask.run(id)
```

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- `conversations` 表已被 `threads` + `thread_messages` 体系取代，但表结构仍保留，存在混淆风险，未来可清理
- `thread_messages` 表以整体 JSON 字符串存储消息列表（非逐条记录），查询效率低，但简化了消息版本管理
- `cleaning_records` 表的 `duration_s` 和 `area_m2` 字段目前插入时均为 `null`，Bridge 尚未返回这两个字段

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **Next.js 热重载单例模式**：通过 `globalThis._db` 保证 dev 模式下热重载不重复创建数据库连接
- **WAL 模式**：`PRAGMA journal_mode = WAL`，提升并发读写性能（适合 Node.js 异步场景）
- **`INSERT OR IGNORE` for threads**：`createThread` 使用幂等写入，避免重复创建线程时报错

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- `better-sqlite3` 是**同步** API，不要在其上层包裹 async/await（直接调用 `.run()` / `.all()` / `.get()`）
- `parseRawTask` 必须在使用 `ScheduledTask.config` 前调用，否则 `config` 是字符串而非对象
- `isQuietHour()` 读取 preferences 表，静默时段默认 22:00 - 7:00，调度器触发时自动跳过
- 数据库文件路径固定为 `process.cwd()/data/home-agent.db`，`data/` 目录不存在时自动创建
