## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

前端 React 组件目录，基于 `@assistant-ui/react` 构建对话界面，使用 Tailwind CSS + shadcn/ui 风格组件。整体 UI 为深色主题（zinc 色系）。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
components/
├── chat/
│   ├── thread.tsx         # 主对话界面（ChatThread 组件，含消息渲染、欢迎屏、输入框）
│   ├── teleop-panel.tsx   # 遥控模式面板：摇杆输入、急停、LiDAR 防撞阻挡提示
│   ├── teleop-mode-store.ts # Teleop 模式全局状态（useSyncExternalStore）
│   ├── tool-uis.tsx       # 各工具的自定义 UI 组件（Tool Call 渲染卡片）
│   └── rough-renderer.tsx # RoughRenderer：Rough.js Canvas 渲染组件，供 DrawSceneToolUI 调用
├── layout/
│   └── sidebar.tsx     # 左侧边栏（机器人状态展示 + 对话历史列表 + 语音记录入口）
├── voice-logs/
│   └── panel.tsx       # VoiceLogsPanel：按天折叠展示语音对话记录（DaySection / SessionItem / SessionDetail）
├── providers.tsx        # 全局 Provider 包裹层（AssistantRuntime + 调度器初始化）
└── ui/                  # shadcn/ui 基础组件（Badge / Button / Card / ScrollArea）
```

**`chat/thread.tsx`** — 对话主界面：
- `ChatThread`：注册所有 Tool UI 组件 + 渲染对话列表
- 集成 `TeleopModeToggle` + `TeleopPanel`：仅在手动遥控模式显示摇杆；关闭/卸载时自动发 stop
- `WelcomeScreen`：空对话时显示，内置 4 个快捷建议（打扫客厅/查看卫生/查看状态/定时清扫）
- 助手显示名来自根 `layout.tsx` 注入的 `AgentDisplayNameProvider`，值由 `AGENT_DISPLAY_NAME`（默认「狗蛋」）决定

**`chat/tool-uis.tsx`** — 工具调用 UI（每个工具一个 React 组件）：
- 已实现的 Tool UI：`RobotStatusToolUI`、`CleanRoomsToolUI`、`FullCleanToolUI`、`PauseToolUI`、`ResumeToolUI`、`ReturnHomeToolUI`、`GetRoomsToolUI`、`TakePhotoToolUI`、`AnalyzeImageToolUI`、`AddTaskToolUI`、`ListTasksToolUI`、`CleaningHistoryToolUI`、`DrawSceneToolUI`
- 未匹配的工具调用使用 `FallbackToolUI`（显示工具名 + 执行状态）；URL 含 `?debug` 时展开 LLM 传参 (args) 和工具返回 (result) 的 JSON 调试面板

**`chat/rough-renderer.tsx`** — Rough.js 画布渲染组件：
- `RoughRenderer`：接收 `DrawScene` 结构（含 width/height/background/commands 等字段），用 `roughjs` 在 Canvas 上绘制手绘风图形
- 支持的图元：rect / circle / ellipse / line / polygon / path / text
- 服务端工具只做 JSON 透传，Canvas 渲染完全在浏览器侧

**`layout/sidebar.tsx`** — 侧边栏：
- `useRobotStatus` hook：轮询 `/api/status`，展示电量、清扫状态、Bridge 连接状态
- 对话历史列表（从 `/api/threads` 获取）+ 新建/删除线程操作
- 语音记录入口按钮（`onOpenVoiceLogs` prop），点击后主区域切换为 `VoiceLogsPanel`

**`voice-logs/panel.tsx`** — 语音记录面板：
- `VoiceLogsPanel`：从 `/api/voice-logs` 获取有记录的天列表，按天折叠展示
- `DaySection`：点击展开时懒加载该天的 session 列表（`/api/voice-logs/sessions?day=`）
- `SessionItem`：显示对话时间/时长/消息数，点击展开具体消息（`SessionDetail`）
- `SessionDetail`：从 `/api/voice-logs/sessions?session_id=` 加载消息，左右气泡布局

**`providers.tsx`** — 全局初始化：
- 包裹 `AssistantRuntimeProvider`（assistant-ui 运行时）
- 在 `useEffect` 中调用 `setTaskExecutor` + `loadScheduledTasks`，完成调度器初始化

**`chat/teleop-panel.tsx`** — 手动遥控 UI：
- 圆形摇杆输出 `throttle/steer`（-1..1）并以 20Hz（50ms）频率发送到 `/api/teleop`
- 展示 LiDAR 安全阻挡文案（too_close / stale / unavailable）
- 提供独立“急停”按钮（立即下发 0 轴命令）

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- `tool-uis.tsx` 文件较长，所有 Tool UI 堆在一个文件，可按功能域拆分（roborock/camera/scheduler 子文件）
- 侧边栏机器人状态刷新依赖手动触发（按钮），没有自动轮询

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **assistant-ui/react**：使用 `@assistant-ui/react` 的 Primitive 组件（`ThreadPrimitive`、`MessagePrimitive` 等）而非自行实现对话状态管理，减少状态管理复杂度
- **深色主题**：全局使用 zinc 色系深色背景（`bg-zinc-950` / `bg-zinc-900`），与 shadcn/ui 的 dark 模式一致
- **FallbackToolUI 调试模式**：通过 `typeof window !== "undefined" && new URLSearchParams(window.location.search).has("debug")` 检测 URL 中是否含 `?debug`，无需 Suspense 包裹。读取 `window.location.search` 而非 `useSearchParams`，是因为此组件嵌套在 assistant-ui 内部，加 Suspense 边界成本高，且调试信息不需要 SSR 精确。

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- 新增 Agent 工具时，必须同时在 `tool-uis.tsx` 中添加对应的 Tool UI 组件，并在 `thread.tsx` 的 `ChatThread` 中注册，否则工具调用只会显示 `FallbackToolUI`
- `providers.tsx` 中的调度器初始化只应执行一次；`useEffect` 依赖数组为 `[]`，确保幂等
- `ui/` 目录下的组件是标准 shadcn/ui 组件，直接复用，不要修改其内部实现
