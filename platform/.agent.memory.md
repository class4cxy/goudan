## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

Python FastAPI 硬件平台服务，监听端口 **8001**。将硬件能力（Roborock 扫地机、电机底盘、摄像头、蓝牙扬声器、LD06 激光雷达、SLAM 建图）封装为本地 REST API + WebSocket，供 Node.js Agent 层调用。

**Chat 模式架构**：麦克风模块已移除（录音由手机端负责），TTS 通过蓝牙 A2DP 外放（SPEAKER_BACKEND=pulseaudio + BluetoothManager 自动设置 PulseAudio default sink）。

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- **SearXNG 需独立启动**：`/search` 接口依赖 Docker 容器（`docker compose up -d`），Platform 本身不管理其生命周期；若容器未启动，`/search` 返回 502 错误
- **searxng/settings.yml 中 `secret_key` 为占位值**：生产/树莓派部署前应替换为随机字符串（`python3 -c "import secrets; print(secrets.token_hex(32))"`）

- **编码器 EMF 噪声导致距离闭环失效**：电机运行时，EMF 耦合进同侧编码器线缆，产生 3-5ms 持续的噪声跳变。该时长与低速下真实脉冲持续时间相近，软件去抖无法区分。根本解决方案：每条编码器 A/B 线对 GND 焊接 100nF 陶瓷电容（硬件滤波），将噪声跳变压缩到 <0.1ms。
  - 临时方案：`/motor/drive` 支持时间标定模式（`DRIVE_SPEED_MM_PER_SEC > 0` 时启用），以 `时间 = 目标距离 / 标定速度` 控制行驶时长，不依赖编码器。`DRIVE_TURN_DEG_PER_SEC > 0` 时同样切换为时间转向。
  - 用 `motion_distance_test.py` 测量实际速度后填入 `.env` 的 `DRIVE_SPEED_MM_PER_SEC`。
- `main.py` 现已支持通过 `.env` 覆盖编码器与里程计关键参数（`ENCODER_*`、`ODOM_*`）；若升级电机/编码器/轮胎后出现距离或角度偏差，优先用该参数组做标定，不必改代码常量
- **IMU 已切换为 BNO055 UART（MCU-55 + CP2102）**：链路已实测可读 `CHIP_ID=0xA0`。`/motor/drive` 转向闭环优先使用原始 IMU（`sensor=raw_imu`），仅在 IMU 不可用时降级 `odometry_theta`。为抑制 UART 偶发 overrun（0x07）造成的短暂丢帧，`devices/imu.py` 已加入读寄存器可恢复错误自动重试。

- **激光雷达 `/lidar/*` 接口未被 Node.js Agent 工具层封装**：Bridge 已有完整雷达接口和 WebSocket 广播，但 `src/lib/agent/tools/` 尚未添加雷达工具，LLM 暂无法主动查询扫描数据
- **`/clean/stop` 和 `/clean/zone` 未被 Node.js Agent 封装**：Bridge 已有这两个接口，但 `src/lib/agent/tools/roborock.ts` 尚未添加对应工具
- **`/history` 与 Node.js 本地 DB 数据不同步**：Bridge 的 `/history` 读云端数据，Agent 的 `getCleaningHistory` 工具读本地 SQLite，两者是独立数据源
- 摄像头云台控制（`python-miio` + `set_motor`）因 token 获取困难暂未实现，等待小米 IoT 开发者平台审批
- **底盘 `/motor/*` 接口未被 Node.js Agent 工具层封装**：Bridge 已有完整底盘接口，但 `src/lib/agent/tools/` 尚未添加电机控制工具，LLM 暂无法主动驱动车辆行走
- **超声波 `/ultrasonic/*` 接口未被 Node.js Agent 工具层封装**：Bridge 已支持距离读取与 too_close 事件广播，但工具层暂无对应 Tool，LLM 暂无法主动查询前方距离
- **激光雷达懒启动**：服务启动时不自动开雷达。生命周期由意图驱动：`POST /slam/start` 自动启动，`POST /slam/stop` 自动停止；`action.navigate` 触发时 MotorEffector 调用 `POST /lidar/start`，导航结束调用 `POST /lidar/stop`
- **激光雷达 WebSocket 广播频率**：默认每圈广播一次（10Hz），可通过环境变量 `LIDAR_BROADCAST_EVERY=5` 降频至 2Hz，避免 Node.js Agent 处理压力过大
- **SLAM 地图广播频率**：位姿 1Hz / 地图 PNG 0.2Hz，在 `SlamConfig.pose_broadcast_every` 和 `map_broadcast_every` 中配置

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **搜索引擎选用 SearXNG（Docker）而非 Python 包**：用户明确要求，理由是聚合多引擎（Bing/Google/DDG/百度），结果更全面；Platform `/search` 接口通过 HTTP 调用本地 SearXNG，无需在 Python 虚拟环境中安装搜索依赖
- **认证优先级：token 文件 > 密码**：优先读 `.roborock_token.json`，避免明文密码存储。token 文件由 `login_once.py` 生成，使用邮箱验证码登录（无需密码）
- **token 文件失效自动重命名**：检测到 token 失效时，自动将文件重命名为 `.roborock_token.json.expired`，防止反复用失效 token 重试
- **直接指定 base_url 绕过 getUrlByEmail**：`login_once.py` 中直接用 `REGION_URLS` 指定服务器地址，避免新绑定邮箱时云端同步延迟导致查询失败
- **只取第一台支持 v1 协议的设备**：`next((d for d in devices if d.v1_properties), None)`，适配石头 T7 系列
- **AudioEffector speak_end 强回执 + 空闲确认**：`audio_effector.py` 采用双保险：①`AUDIO_SPEAK_END_FALLBACK_MS` 兜底定时器（默认 15000ms），播放链路异常时强制广播 `sense.audio.speak_end`（`payload.forced=true`）；②`AUDIO_SPEAK_END_IDLE_CONFIRM_MS` 空闲确认延迟（默认 250ms），避免句间短暂空窗时过早发出 `speak_end`。
- **移除本地 STT 组件**：Bridge 不再加载 `LocalSTT`，也不再提供 `/stt/*` 接口；语音转写统一由 Node 侧调用云端 ASR 完成，减少启动耗时与本地模型依赖
- **超声波默认引脚固定为 GPIO20/GPIO21**：基于真机扫描结果确定（`ultrasonic_test.py --scan`），可通过环境变量 `ULTRASONIC_TRIG_PIN` / `ULTRASONIC_ECHO_PIN` 覆盖
- **运动标定参数改为代码内固定（2026-04）**：按当前 `encoder_accuracy_test.py` 实测在 `main.py` 写死底盘与里程计参数：`left_scale=1.00`、`right_scale=0.99`、`lines_per_rev=140`、`wheel_radius_mm=34.0`、`wheel_base_mm=160.0`。当前驱动轮为直径 68mm 的麦克姆轮；用户明确要求“不要通过 .env 调参”。
- **`/motor/drive` 与测试参数同步（2026-04）**：闭环直行默认速度固定为 `40`，当前不再做“接近终点二次降速”。原因：这套距离标定是按固定速度测出来的，中途改为两段速度会引入新的速度相关误差，导致严重缩水。停车判据与 `encoder_accuracy_test.py` 保持一致，统一使用 `odometry.get_and_reset_travel()` 的累计路程。左右轮累计距离与对称比仅用于结束后的日志诊断，不参与停车判断。
- **串口端口策略（2026-04）**：代码层不再依赖 `.env` 覆盖；LD06 固定 `/dev/ttyUSB1`。BNO055 优先 `/dev/ttyUSB0`，若握手失败自动扫描 `ttyUSB* / ttyACM*` 选择可用端口，降低 USB 枚举顺序漂移引发的 IMU 掉线风险
- **转向闭环 IMU 符号自校正（2026-04）**：`/motor/drive` 的 raw_imu 路径在每次转向开始时，根据首个有效 `gyro_z` 自动校正符号方向，避免更换 IMU 或安装方向后出现“actual=0、opposite 很大、超时失败”。
- **转向闭环 IMU 低贡献二段兜底（2026-04）**：`/motor/drive` 的 angle 路径会先走 raw_imu 第一阶段；若起转约 1.2s 内角速度贡献偏低，提前切换 `odometry_theta_fallback` 继续转向，并在需要时追加短暂补偿窗口，降低“IMU在线但积分几乎不增长”导致的超时失败。
- **SLAM 里程计角度语义修正（2026-04）**：`odometry.get_velocity_for_slam()` 传给 SLAM/AMCL 的 `dtheta` 必须保留符号，不能取绝对值；否则左/右转会被折叠成同向初始猜测，建图易出现重影、放射状毛刺和异常角度累计。`slam_engine.py` 同时将 `theta_deg` 归一化到 `[0, 360)`，避免 `/slam/pose` 与前端显示 `815°` 这类累计角度。
- **运动传感器启动竞态自恢复（2026-04）**：`main.py` 在启动阶段若检测到 `encoder` / `imu` 处于 simulation，会创建后台任务每 5 秒重试初始化；`/motor/drive` 执行前也会按需重试，缓解 PM2 冷启动时 USB/GPIO 枚举延迟导致“首次失败后整段会话不可用”。
- **底盘模拟模式改为显式失败（2026-04）**：`/motor/command` 与 `/motor/drive` 在 `chassis.is_simulation` 时直接返回 503，不再静默执行“API 成功但车不动”的假动作，便于快速定位 GPIO 依赖/权限/启动用户问题。
- **手动遥控会话化 + 按需 watchdog（2026-04）**：新增 `/teleop/*`。默认聊天态不常驻 watchdog；仅 `teleop/start` 后启用超时刹停（默认 300ms），`teleop/stop` 或服务退出会强制停车并关闭 watchdog。
- **手动遥控双向 LiDAR 防撞（2026-04）**：`/teleop/command` 按运动方向检查扇区：前进看 0°、后退看 180°。若最近距离小于 `min_safe_mm`（默认 350mm）则立即刹停并拒绝该方向运动。仅在 `stale/no_scan` 时返回“数据不可用”并阻断；`no_points` 不再误判为 stale。
- **遥控默认速度下调（2026-04）**：`/teleop/start` 的 `max_speed` 默认从 60 下调到 30（约慢一半），并将 `scan_freshness_ms` 默认放宽至 2000ms，减少低频抖动下的误阻断。
- **临时关闭 Teleop LiDAR 防撞（用户要求，2026-04）**：`/teleop/start` 新增 `lidar_safety_enabled` 开关，默认关闭（`TELEOP_LIDAR_SAFETY_ENABLED=0`）。关闭时遥控不再因雷达误判阻断；如需恢复，仅在 start 请求传 `lidar_safety_enabled=true` 或配置环境变量启用。
