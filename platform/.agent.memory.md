## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

Python FastAPI 硬件平台服务，监听端口 **8001**。将硬件能力（Roborock 扫地机、电机底盘、摄像头、蓝牙扬声器、LD06 激光雷达、SLAM 建图）封装为本地 REST API + WebSocket，供 Node.js Agent 层调用。

**Chat 模式架构**：麦克风模块已移除（录音由手机端负责），TTS 通过蓝牙 A2DP 外放（SPEAKER_BACKEND=pulseaudio + BluetoothManager 自动设置 PulseAudio default sink）。

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- **音乐音量调整对当前曲目不生效**：`MusicPlayer.set_volume()` 修改的是下一次 ffplay 进程启动时的 `-af volume=` 参数，已在播放的曲目不受影响，需要跳到下一曲后才能生效
- **暂停（SIGSTOP）会引起短暂音频点击声**：ffplay 被冻结后 PulseAudio 缓冲区耗尽，恢复（SIGCONT）时有破音；对于临时 TTS 闪避这是可接受的，若需无缝暂停可改用 mpv IPC 方案
- **yt-dlp 解析需联网且速度慢**：每次播放新歌前需先执行 yt-dlp 查询（最多 30s），LOADING 状态期间 Agent 应给用户反馈"正在搜索"
- **SearXNG 需独立启动**：`/search` 接口依赖 Docker 容器（`docker compose up -d`），Platform 本身不管理其生命周期；若容器未启动，`/search` 返回 502 错误
- **searxng/settings.yml 中 `secret_key` 为占位值**：生产/树莓派部署前应替换为随机字符串（`python3 -c "import secrets; print(secrets.token_hex(32))"`）

- **激光雷达 `/lidar/*` 接口未被 Node.js Agent 工具层封装**：Bridge 已有完整雷达接口和 WebSocket 广播，但 `src/lib/agent/tools/` 尚未添加雷达工具，LLM 暂无法主动查询扫描数据
- **`/clean/stop` 和 `/clean/zone` 未被 Node.js Agent 封装**：Bridge 已有这两个接口，但 `src/lib/agent/tools/roborock.ts` 尚未添加对应工具
- **`/history` 与 Node.js 本地 DB 数据不同步**：Bridge 的 `/history` 读云端数据，Agent 的 `getCleaningHistory` 工具读本地 SQLite，两者是独立数据源
- 摄像头云台控制（`python-miio` + `set_motor`）因 token 获取困难暂未实现，等待小米 IoT 开发者平台审批
- **底盘 `/motor/*` 接口未被 Node.js Agent 工具层封装**：Bridge 已有完整底盘接口，但 `src/lib/agent/tools/` 尚未添加电机控制工具，LLM 暂无法主动驱动车辆行走
- **超声波 `/ultrasonic/*` 接口未被 Node.js Agent 工具层封装**：Bridge 已支持距离读取与 too_close 事件广播，但工具层暂无对应 Tool，LLM 暂无法主动查询前方距离
- **激光雷达懒启动**：服务启动时不自动开雷达。生命周期由意图驱动：`POST /slam/start` 自动启动，`POST /slam/stop` 自动停止；`action.navigate` 触发时 MotorEffector 调用 `POST /lidar/start`，导航结束调用 `POST /lidar/stop`
- **激光雷达 WebSocket 广播频率**：默认每圈广播一次（10Hz），可通过环境变量 `LIDAR_BROADCAST_EVERY=5` 降频至 2Hz，避免 Node.js Agent 处理压力过大
- **SLAM 地图广播频率**：位姿 1Hz / 地图 PNG 0.2Hz，在 `SlamConfig.pose_broadcast_every` 和 `map_broadcast_every` 中配置

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **搜索引擎选用 SearXNG（Docker）而非 Python 包**：用户明确要求，理由是聚合多引擎（Bing/Google/DDG/百度），结果更全面；Platform `/search` 接口通过 HTTP 调用本地 SearXNG，无需在 Python 虚拟环境中安装搜索依赖
- **认证优先级：token 文件 > 密码**：优先读 `.roborock_token.json`，避免明文密码存储。token 文件由 `login_once.py` 生成，使用邮箱验证码登录（无需密码）
- **token 文件失效自动重命名**：检测到 token 失效时，自动将文件重命名为 `.roborock_token.json.expired`，防止反复用失效 token 重试
- **直接指定 base_url 绕过 getUrlByEmail**：`login_once.py` 中直接用 `REGION_URLS` 指定服务器地址，避免新绑定邮箱时云端同步延迟导致查询失败
- **只取第一台支持 v1 协议的设备**：`next((d for d in devices if d.v1_properties), None)`，适配石头 T7 系列
- **AudioEffector speak_end 强回执 + 空闲确认**：`audio_effector.py` 采用双保险：①`AUDIO_SPEAK_END_FALLBACK_MS` 兜底定时器（默认 15000ms），播放链路异常时强制广播 `sense.audio.speak_end`（`payload.forced=true`）；②`AUDIO_SPEAK_END_IDLE_CONFIRM_MS` 空闲确认延迟（默认 250ms），避免句间短暂空窗时过早发出 `speak_end`。
- **移除本地 STT 组件**：Bridge 不再加载 `LocalSTT`，也不再提供 `/stt/*` 接口；语音转写统一由 Node 侧调用云端 ASR 完成，减少启动耗时与本地模型依赖
- **超声波默认引脚固定为 GPIO20/GPIO21**：基于真机扫描结果确定（`ultrasonic_test.py --scan`），可通过环境变量 `ULTRASONIC_TRIG_PIN` / `ULTRASONIC_ECHO_PIN` 覆盖

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- 启动前**无需**手动激活虚拟环境：`package.json` 的 `platform` 脚本已直接调用 `.venv/bin/uvicorn`
- 启动命令（npm）：`npm run platform`（等价于 `cd platform && .venv/bin/uvicorn main:app --host 0.0.0.0 --port 8001 --reload`）
- 如果 `.venv` 损坏（例如项目目录被重命名/移动），需重建虚拟环境：`rm -rf .venv && python3.11 -m venv .venv && pip install -r requirements.txt`（跳过 breezyslam 报错）
- 首次使用需先运行 `python login_once.py` 生成 token 文件
- token 文件权限为 `0o600`（仅当前用户可读），不要修改权限
- 如果 token 失效，删除 `.roborock_token.json.expired` 并重新运行 `login_once.py`
- 该目录已安装了 `python-miio` 和 `micloud`（用于探索摄像头控制），不属于 `requirements.txt` 正式依赖
- 超声波阈值与轮询频率可通过环境变量配置：`ULTRASONIC_TOO_CLOSE_CM`、`ULTRASONIC_POLL_INTERVAL`
