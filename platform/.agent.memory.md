## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

Python FastAPI 硬件平台服务，监听端口 **8001**。将硬件能力（Roborock 扫地机、电机底盘、摄像头、麦克风、扬声器、LD06 激光雷达、SLAM 建图）封装为本地 REST API + WebSocket，供 Node.js Agent 层调用。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

**已开放的 REST 接口：**

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/health` | 健康检查（设备就绪状态、房间列表、token 文件状态） |
| GET | `/status` | 机器人实时状态（电量、清扫状态、错误码等） |
| GET | `/rooms` | 房间列表（名称 ↔ segment_id 双向映射） |
| POST | `/clean/start` | 全屋清扫 |
| POST | `/clean/rooms` | 指定房间清扫（body: room_names/room_ids/repeat） |
| POST | `/clean/zone` | 区域清扫（body: zones 坐标矩形数组/repeat） |
| POST | `/clean/pause` | 暂停清扫 |
| POST | `/clean/resume` | 继续清扫 |
| POST | `/clean/stop` | 停止清扫（原地停止，不回充） |
| POST | `/home` | 停止并回充电桩 |
| GET | `/history` | 从 Roborock 云端拉取清洁历史（query param: limit） |
| POST | `/motor/command` | 底盘运动指令（body: command/speed/duration） |
| POST | `/motor/set` | 单电机控制（body: position/direction/speed） |
| GET | `/motor/status` | 查询各电机当前速度快照 |
| GET | `/lidar/status` | 激光雷达连接状态（port/is_simulation/completed_scans/latest_scan 摘要） |
| GET | `/lidar/scan` | 最新一圈全量扫描数据（所有测距点，含角度/距离/置信度） |
| GET | `/lidar/scan/valid` | 最新一圈有效点（已过滤无效反射，适合地图构建） |
| POST | `/slam/start` | 初始化 SLAM 引擎，开始建图 |
| POST | `/slam/stop` | 停止建图（冻结地图） |
| POST | `/slam/reset` | 重置 SLAM（清空地图和位姿） |
| GET | `/slam/status` | SLAM 引擎状态（is_mapping/scan_count/pose/地图参数） |
| GET | `/slam/pose` | 当前机器人位姿（x_mm, y_mm, theta_deg） |
| GET | `/slam/map` | 当前地图 PNG base64 + robot_pixel + 元数据 |
| POST | `/slam/save` | 保存地图到 platform/maps/（.pgm + .json） |
| GET | `/slam/maps` | 列出所有已保存地图 |
| POST | `/slam/load/{map_name}` | 加载已保存的地图 |

**WebSocket 广播事件（Bridge → Node.js，感官事件）：**

| type | payload 字段 | 说明 |
|------|-------------|------|
| `sense.lidar.scan` | `timestamp_ms`, `rpm`, `point_count`, `valid_count`, `points[]` | 每圈扫描完成时广播（频率由 `LIDAR_BROADCAST_EVERY` 控制） |
| `sense.slam.pose` | `x_mm`, `y_mm`, `theta_deg`, `scan_count` | 机器人位姿（建图中，每 10 圈 ≈ 1Hz） |
| `sense.slam.map_update` | `image_b64`, `width`, `height`, `mm_per_pixel`, `robot_pixel`, `pose`, `scan_count` | 地图 PNG 更新（建图中，每 50 圈 ≈ 0.2Hz） |

**WebSocket 支持的行动指令（Node.js → Bridge）：**

| type | payload 字段 | 说明 |
|------|-------------|------|
| `action.speak` | `text`, `interrupt_current` | TTS 播报 |
| `action.mute` | — | 麦克风静音 |
| `action.unmute` | — | 麦克风取消静音 |
| `action.motor` | `command`, `speed?`, `duration?` | 底盘运动指令 |

**全局状态（`state` dict）：**
- `device_manager`：DeviceManager 实例
- `device`：RoborockDevice（取第一台支持 v1 协议的设备）
- `rooms`：`{room_name: segment_id}`
- `room_ids`：`{segment_id: room_name}`
- `ready`：bool，设备是否就绪

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- **激光雷达 `/lidar/*` 接口未被 Node.js Agent 工具层封装**：Bridge 已有完整雷达接口和 WebSocket 广播，但 `src/lib/agent/tools/` 尚未添加雷达工具，LLM 暂无法主动查询扫描数据
- **`/clean/stop` 和 `/clean/zone` 未被 Node.js Agent 封装**：Bridge 已有这两个接口，但 `src/lib/agent/tools/roborock.ts` 尚未添加对应工具
- **`/history` 与 Node.js 本地 DB 数据不同步**：Bridge 的 `/history` 读云端数据，Agent 的 `getCleaningHistory` 工具读本地 SQLite，两者是独立数据源
- 摄像头云台控制（`python-miio` + `set_motor`）因 token 获取困难暂未实现，等待小米 IoT 开发者平台审批
- **底盘 `/motor/*` 接口未被 Node.js Agent 工具层封装**：Bridge 已有完整底盘接口，但 `src/lib/agent/tools/` 尚未添加电机控制工具，LLM 暂无法主动驱动车辆行走
- **激光雷达 WebSocket 广播频率**：默认每圈广播一次（10Hz），可通过环境变量 `LIDAR_BROADCAST_EVERY=5` 降频至 2Hz，避免 Node.js Agent 处理压力过大
- **SLAM 地图广播频率**：位姿 1Hz / 地图 PNG 0.2Hz，在 `SlamConfig.pose_broadcast_every` 和 `map_broadcast_every` 中配置

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **认证优先级：token 文件 > 密码**：优先读 `.roborock_token.json`，避免明文密码存储。token 文件由 `login_once.py` 生成，使用邮箱验证码登录（无需密码）
- **token 文件失效自动重命名**：检测到 token 失效时，自动将文件重命名为 `.roborock_token.json.expired`，防止反复用失效 token 重试
- **直接指定 base_url 绕过 getUrlByEmail**：`login_once.py` 中直接用 `REGION_URLS` 指定服务器地址，避免新绑定邮箱时云端同步延迟导致查询失败
- **只取第一台支持 v1 协议的设备**：`next((d for d in devices if d.v1_properties), None)`，适配石头 T7 系列

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- 启动前**无需**手动激活虚拟环境：`package.json` 的 `platform` 脚本已直接调用 `.venv/bin/uvicorn`
- 启动命令（npm）：`npm run platform`（等价于 `cd platform && .venv/bin/uvicorn main:app --host 0.0.0.0 --port 8001 --reload`）
- 如果 `.venv` 损坏（例如项目目录被重命名/移动），需重建虚拟环境：`rm -rf .venv && python3.11 -m venv .venv && pip install -r requirements.txt`（跳过 breezyslam 报错）
- 首次使用需先运行 `python login_once.py` 生成 token 文件
- token 文件权限为 `0o600`（仅当前用户可读），不要修改权限
- 如果 token 失效，删除 `.roborock_token.json.expired` 并重新运行 `login_once.py`
- 该目录已安装了 `python-miio` 和 `micloud`（用于探索摄像头控制），不属于 `requirements.txt` 正式依赖
