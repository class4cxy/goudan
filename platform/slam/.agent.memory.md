## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

`bridge/slam/` 是 SLAM（同时定位与建图）算法层，封装 breezyslam 库，将 LD06 激光雷达原始扫描数据转化为占用栅格地图和机器人位姿。是建图→分区→导航链路的核心计算模块，纯算法层，不含 WebSocket / FastAPI 任何网络逻辑。

## 文件夹内容结构

> 描述主要文件/子目录的功能、关键类或函数。

```
bridge/slam/
├── __init__.py          # 包入口，导出 SlamEngine / SlamConfig / DEFAULT_SLAM_CONFIG
├── slam_engine.py       # SlamEngine 核心类（breezyslam RMHC_SLAM 封装）
└── .agent.memory.md     # 本文件
```

**SlamEngine 核心接口：**

| 方法/属性 | 说明 |
|-----------|------|
| `start_mapping() → bool` | 初始化 RMHC_SLAM，开始接受扫描帧；返回 False 表示 breezyslam 未安装 |
| `stop_mapping()` | 冻结地图，停止更新 |
| `reset()` | 完全清空地图和位姿 |
| `process_scan(LidarScan)` | 喂入一圈扫描数据（在串口线程调用，线程安全） |
| `get_pose() → (x_mm, y_mm, theta_deg)` | 获取当前机器人位姿（线程安全） |
| `pose_to_pixel(x_mm, y_mm) → (px, py)` | mm 坐标 → 像素坐标转换 |
| `get_map_png_b64(draw_robot=True) → str` | 渲染当前地图为 PNG base64（用 cv2） |
| `save_map(name="") → dict` | 保存地图到 bridge/maps/（.pgm + .json） |
| `load_map(name) → bool` | 从 bridge/maps/ 加载已保存地图 |
| `list_maps() → list[dict]` | 列出 bridge/maps/ 下所有地图元数据 |
| `status → dict` | 引擎状态摘要（is_mapping/scan_count/pose/地图参数） |

**SlamConfig 关键参数：**

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `map_size_pixels` | 500 | 地图分辨率（500×500格） |
| `map_size_meters` | 10.0 | 覆盖物理范围（10m×10m） |
| `mm_per_pixel` | 20.0 | 精度：每像素代表 20mm |
| `map_quality` | 50 | 地图更新强度（值越大更新越快但噪声越多） |
| `sigma_xy_mm` | 100 | 位置不确定度（值越大粒子扩散越广） |
| `sigma_theta_deg` | 20 | 角度不确定度 |
| `pose_broadcast_every` | 10 | 每 10 圈触发 on_update（→1Hz 位姿广播） |
| `map_broadcast_every` | 50 | 每 50 圈触发地图广播（→0.2Hz） |

**地图字节语义（breezyslam 约定）：**
- `0`      = 未探索（灰色）
- `1–127`  = 障碍物（值越小置信越高，渲染越黑）
- `128–255` = 可通行（值越大置信越高，渲染越白）

**扫描重采样流程（`_resample_scan`）：**
LD06 每圈 ~450 个非均匀角度点 → 360 个均匀 1°/点数组
- 每 1° 桶取最近（最小距离）有效点
- 无读数的桶填 0（breezyslam 解释为"无探测"）

**地图文件格式（bridge/maps/）：**
- `{name}.pgm`  — P5 二进制灰度图（ROS 兼容），可直接被 nav2 加载
- `{name}.json` — 元数据（尺寸/精度/扫描次数/保存时位姿/时间戳）

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- **无里程计（Odometry）输入**：当前 `process_scan()` 调用 `slam.update(distances)` 不传 velocities，完全依赖扫描匹配估算位移。走廊等特征稀少区域旋转漂移风险较高。后续加 MPU6050 陀螺仪后，可改为 `slam.update(distances, velocities=(dxy, dtheta, dt))`
- **地图尺寸固定**：10m×10m，大房子可能不够，需调大 `map_size_meters`（会增加内存和渲染时间）
- **breezyslam 版本兼容性**：breezyslam C 扩展在树莓派上需要 gcc 编译。若编译失败会自动降级为纯 Python（速度约慢 10×），需测试树莓派 5 上的实际性能

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **选用 breezyslam 而非 ROS2 hector_slam**：用户明确不引入 ROS2，breezyslam 是纯 Python 可 pip 安装的最轻量 SLAM 库，与现有 FastAPI Bridge 架构无缝集成
- **地图分区采用手动标注（用户决策）**：算法分区（Voronoi/FloodFill）无法准确识别功能区域（客厅 vs 餐厅），用户将在 Web UI 中手动框选并命名房间
- **`on_update` 回调在串口线程同步执行**：为了简化跨线程调度，SLAM 更新完成后在串口线程同步触发回调（LidarSensor 内部用 `run_coroutine_threadsafe` 转至 asyncio 线程），不增加额外线程
- **PNG 渲染 offload 到 asyncio 线程池**：`get_map_png_b64()` 是 CPU 密集型，在 REST 接口中用 `asyncio.to_thread()` 调用，避免阻塞 asyncio 事件循环

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- **`process_scan` 必须在 `start_mapping()` 之后才有效**：未 start 时调用会静默忽略（is_mapping=False）
- **`get_map_png_b64` 依赖 opencv**：若 `opencv-python-headless` 未安装，返回 None 而非异常
- **`save_map` 地图为空时返回 None**：调用方需检查返回值
- **坐标系**：x 轴向右，y 轴向下（图像坐标系）；机器人启动位置在地图中心 (250,250)；x_mm/y_mm 正值对应右下方向
- **`map_size_pixels` 修改需同步修改 `_map_bytes`**：bytearray 大小是 `map_size_pixels²`，若运行时修改配置会导致越界
