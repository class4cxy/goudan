## 文件夹摘要

> 描述本文件夹在工程中的位置、定位与作用。

`platform/devices/` 是机器车硬件能力的抽象层，是 Platform 服务中唯一直接操作硬件的包。目前已实现：4 轮 DC 电机底盘（Chassis）、摄像头双轴云台（CameraMount）、摄像头拍照（Camera）、麦克风 VAD 采集（Microphone）、TTS 扬声器播放（Speaker）、LD06 激光雷达（Lidar，CP2102 USB-TTL 串口）、INA219 电源传感器（PowerSensor，I2C）。后续可在此继续扩展超声波、RGB 灯控等。

**底盘核心类：**

| 类/常量 | 职责 |
|---------|------|
| `MotorPins(in1, in2, en=-1)` | 单电机 GPIO 引脚配置（en=-1=无外部EN，直接IN-PWM模式） |
| `Motor(position, pins)` | 单路 DC 电机：`forward(speed)` / `backward(speed)` / `stop()` |
| `ChassisConfig` | 四路电机引脚 + `default_speed` |
| `DEFAULT_CONFIG` | MAKEROBO 实测引脚（M1:24/25, M2:27/26, M3:5/6, M4:22/9） |
| `Chassis(config)` | 4轮底盘：整车运动 + 单电机控制 + 定时指令 + 状态查询 |

**云台核心类：**

| 类/常量 | 职责 |
|---------|------|
| `ServoConfig(pin, min_angle, max_angle, default_angle)` | 单轴舵机配置 |
| `CameraConfig(pan, tilt)` | 双轴云台配置 |
| `DEFAULT_CAMERA_CONFIG` | MAKEROBO 实测（Pan GPIO12 0°–180°，Tilt GPIO13 75°–105°） |
| `Servo(name, config)` | 单轴舵机：`set_angle(angle)` / `move_by(delta)` / `center()` |
| `CameraMount(config)` | 双轴云台：`pan_to/tilt_to/pan_by/tilt_by/look_at/center/sweep_pan/sweep_tilt` |

**摄像头采集核心类：**

| 类/常量 | 职责 |
|---------|------|
| `CaptureConfig(source, width, height, jpeg_quality, snapshot_dir)` | 采集参数配置 |
| `DEFAULT_CAPTURE_CONFIG` | 默认（USB source=0，640×480，质量85，保存至 /tmp/roborock_snapshots） |
| `Camera(config)` | 拍照控制器：`capture()` / `capture_base64()` / `capture_to_file()` / `cleanup()` |

**Camera 主要方法：**

```python
cam = Camera()                          # 使用默认配置，也可传自定义 CaptureConfig
jpeg_bytes = cam.capture()              # → bytes | None，首次调用自动 open()
b64_str    = cam.capture_base64()       # → str | None，base64 编码 JPEG（嵌入 JSON）
path       = cam.capture_to_file()      # → str | None，按时间戳命名保存到 snapshot_dir
cam.status                              # → dict（source、is_open、cv2_available 等）
cam.cleanup()                           # 释放 cv2.VideoCapture 资源
```

**Chassis 主要方法：**

```python
# 整车运动（同步）
chassis.forward(speed=70)
chassis.backward(speed=70)
chassis.turn_left(speed=60)   # 原地左旋：左轮反转，右轮正转
chassis.turn_right(speed=60)  # 原地右旋：右轮反转，左轮正转
chassis.stop()

# 定时运动（异步，duration 秒后自动停止）
await chassis.execute_timed("forward", speed=60, duration=2.0)

# 单电机精细控制
chassis.set_motor("front_left", "backward", speed=50)

# 状态查询
chassis.status        # → {"front_left": 60, "front_right": 60, ...}
chassis.is_simulation # → bool

# 资源清理（程序退出前必须调用）
chassis.cleanup()
```

**GPIO 引脚实测映射（BCM 编号，GPIO 探针 Phase A 实机确认）：**

| 位置 | 接口 | IN1 | IN2 | EN | 说明 |
|------|------|-----|-----|----|------|
| front_left  | M1 | 24 | 25 | -1 | Phase A 确认 |
| front_right | M2 | 27 | 26 | -1 | Phase A 确认 |
| rear_left   | M3 | 5  | 6  | -1 | Phase A 确认 |
| rear_right  | M4 | 22 | 9  | -1 | 全部实测确认（GPIO9 = SPI_MISO 复用） |

EN=-1：MAKEROBO SW-6008 驱动芯片的 EN 引脚已内部接高电平，无需外部 GPIO 控制。速度通过对 IN1/IN2 引脚直接 PWM 实现（直接 IN PWM 模式）。

**激光雷达核心类：**

| 类/常量 | 职责 |
|---------|------|
| `LidarConfig(port, baud_rate, timeout, broadcast_every_n_scans)` | 串口配置；port 默认 `/dev/ttyUSB0`，baud_rate 固定 230400 |
| `DEFAULT_LIDAR_CONFIG` | 默认配置（可被环境变量 `LIDAR_PORT` 覆盖） |
| `LidarPoint(angle, distance, confidence)` | 单个测距点；`is_valid` = 20mm≤dist≤12000mm 且 confidence>10 |
| `LidarScan(timestamp_ms, rpm, points)` | 一圈完整扫描；`valid_points` 过滤无效点；`to_dict()` 序列化 |
| `Lidar(config, on_scan)` | 主控制器；`start()` 打开串口+启动线程，`stop()` 关闭；非树莓派自动降级 |

**Lidar 主要属性/方法：**

```python
lidar = Lidar(LidarConfig(port="/dev/ttyUSB0"), on_scan=callback)
lidar.start()              # 打开串口，启动后台线程（非阻塞）
lidar.stop()               # 停止线程，关闭串口
lidar.is_simulation        # → bool（True = 串口不可用，模拟模式）
lidar.is_running           # → bool（读取线程是否存活）
lidar.latest_scan          # → LidarScan | None（线程安全，加锁）
lidar.status               # → dict（port/is_simulation/is_running/completed_scans/latest_scan摘要）
```

**LD06 协议摘要：**
- 接口：CP2102 USB-TTL → 树莓派 USB 2.0
- 接线：LD06 P5V→5V，GND→GND，Tx→RXD（单向接收，PWM 悬空=内部调速 10Hz）
- 数据包：47 字节，头 0x54，VerLen 0x2C，每包 12 个测距点，CRC8（多项式 0x4D）
- 测量频率：4500Hz；扫描频率：10Hz；每圈约 450 点
- 有效范围：20mm–12000mm；角度精度：0.01°

**音频核心类：**

| 类/参数 | 职责 |
|---------|------|
| `Microphone(on_speech_start, on_speech_end, vad_aggressiveness, device)` | 麦克风 VAD 采集器；通过回调推送语音事件，不含网络逻辑 |
| `Microphone.mute()` / `unmute()` | 外放时静音防回声；上层（AudioSensor）在播放时调用 |

**电源传感器核心类：**

| 类/常量 | 职责 |
|---------|------|
| `PowerSensorConfig(shunt_ohms, i2c_address, max_expected_amps, poll_interval_s, battery_full_v, battery_empty_v, low_battery_pct)` | INA219 配置；默认 2S LiPo（8.4V 满 / 6.6V 空），低电量阈值 20% |
| `PowerReading(timestamp_ms, voltage_v, current_ma, power_mw, shunt_mv, battery_pct, is_charging)` | 单次采样数据；`battery_pct` 由电压线性估算，`is_charging` = current_ma < -50mA |
| `PowerSensor(config, on_reading, on_low_battery)` | 主控制器；`start()` 启动 I2C 轮询线程，`stop()` 关闭；非树莓派或 INA219 未接自动降级 |

```python
sensor = PowerSensor(
    config=PowerSensorConfig(poll_interval_s=5, low_battery_pct=20),
    on_reading=lambda r: ...,      # 每次采样回调
    on_low_battery=lambda r: ...,  # 低电量回调（充电中不触发，节流 60s）
)
sensor.start()
sensor.latest_reading   # → PowerReading | None（线程安全）
sensor.status           # → dict（含 battery_pct / is_charging / is_low_battery）
sensor.stop()
```

**INA219 接线（已实测）：**
- VCC → 树莓派 3.3V（Pin 1）；GND → Pin 6；SDA → GPIO2（Pin 3）；SCL → GPIO3（Pin 5）
- `busnum=1` 必须显式传入（树莓派 5 无法自动识别硬件版本）
- 绿色接线柱 Vin+ / Vin- 串联在供电回路中（正极→Vin+，负极侧→Vin-）

## 遗留问题

> 记录暂时性妥协、技术债或未完成的优化。

- **PowerSensor 电量百分比为线性估算**：实际 LiPo 放电曲线非线性，低电量段误差较大（约 ±5–10%）。如需精确电量，需查 LiPo 放电曲线表做分段插值，当前线性算法够用于告警判断。
- **只支持原地差速转向**：当前 `turn_left` / `turn_right` 均为原地旋转（左右轮速相反）。如需弧形转向（一侧慢速、另一侧快速），需额外实现 `arc_left(inner_speed, outer_speed)` 方法。
- **Camera 首帧质量问题**：USB 摄像头刚打开时前几帧可能较暗或模糊（感光元件未稳定），`capture()` 已通过 `grab()`（丢弃一帧）缓解，但若图像质量仍差，可在 `open()` 后 sleep 0.5s 再读帧。
- **cv2 未安装降级**：若 `opencv-python-headless` 未安装，`Camera` 实例仍可正常创建，所有 `capture*()` 方法返回 `None`，`status["cv2_available"] = False`，服务不会崩溃。
- **单例实例**：`main.py` 在模块级创建全局 `chassis = Chassis()`，多次 import 不会重复初始化 GPIO，但若需要多底盘支持则需重构。
- **PWM_FREQ 固定为 1000Hz**：写死在 `Motor` 类中，不在 `ChassisConfig` 内，如需修改需改 `Motor.PWM_FREQ`。
- **GPIO 17 = 蜂鸣器**：MAKEROBO 板载蜂鸣器由 GPIO 17 控制，设为 HIGH 时蜂鸣，勿用于电机控制。
- **GPIO 9 = M4 in2**：SPI_MISO 引脚复用为 M4 右后轮反转控制，说明该板 SPI 总线未使用。

## 特殊决策

> 记录非标准实现的原因、用户指定的特殊要求（含来源）。

- **INA219 `busnum=1` 必须显式传入**：树莓派 5 的硬件版本号无法被 pi-ina219 自动识别，不传会抛 `Could not determine default I2C bus for platform`。已在 `power_sensor.py` 和 `power_sensor_test.py` 中固定传入 `busnum=1`。
- **`_FakeGPIO` 降级策略**：在 `gpio_adapter.py` 模块级执行 `import RPi.GPIO`，失败则替换为 `_FakeGPIO` 对象，确保开发机/CI 环境无需任何 mock 配置即可 import 和运行（所有操作以 DEBUG 日志输出）。
- **`Chassis` / `Camera` 在 `main.py` 模块级实例化**：与 `AudioSensor` / `AudioEffector` 保持一致的生命周期管理模式，`_shutdown()` 统一调用 `chassis.cleanup()` 和 `cam.cleanup()`。
- **`Camera` source 支持环境变量 `CAMERA_SOURCE`**：若为纯数字则解析为 USB 设备号，否则视为 URL（RTSP 等）。snapshot_dir 由 `CAMERA_SNAPSHOT_DIR` 控制。
- **`execute_timed` 自动取消旧任务**：每次新指令到来时取消上一个定时停止 Task，避免多条时序指令叠加引起意外停止。
- **速度 `None` → 默认值**：所有运动方法接受 `speed: int | None`，`None` 时使用 `ChassisConfig.default_speed`（60），避免调用侧必须传速度。
- **直接 IN PWM 模式（MAKEROBO 实测）**：SW-6008 驱动芯片的 EN 引脚已板内接高电平，`MotorPins.en=-1` 时 `Motor` 类自动切换为"对 IN1/IN2 分别建立 PWM"的速度控制方式，而非传统 L298N 的 EN PWM 模式。如换用 L298N，传入 `MotorPins(in1=x, in2=y, en=z)` 即可恢复传统模式。

## 注意事项

> 记录隐式约定、相似功能函数的区分与选择指南。

- **`turn_left` vs `set_motor`**：`turn_left` 是差速整车转向（4 个电机联动）；`set_motor` 用于单个电机独立控制（调试、特殊动作）。不要用 `set_motor` 去模拟整车转向。
- **`cleanup()` 必须调用**：`GPIO.cleanup()` 会复位所有引脚，程序异常退出时若不调用可能导致引脚保持高电位，电机继续运转。`main.py` 的 `_shutdown()` 已封装，但手动测试时需注意。
- **树莓派必须以 `gpio` 组或 root 身份运行**：否则 `RPi.GPIO` 会抛 `RuntimeError`，并自动降级为模拟模式（行为上不报错，但不会驱动电机）。
- **扩展新硬件**：在 `bridge/devices/` 内新建模块（如 `ultrasonic.py`），在 `__init__.py` 中补充导出，并更新本文档。对应真机测试脚本命名为 `<设备名>_test.py`，同样放在本目录内。
- **`Microphone` vs `AudioSensor`**：`Microphone` 是纯硬件层（VAD+采集，无网络），`AudioSensor`（`bridge/audio_sensor.py`）是应用层（Microphone → WebSocket 广播）。直接测试麦克风用 `Microphone`，不要在测试中引入 `AudioSensor`。
- **`Speaker` vs `AudioEffector`**：同上，`Speaker` 是纯硬件层（TTS+播放），`AudioEffector`（`bridge/audio_effector.py`）是应用层（WebSocket 指令 → Speaker 队列）。
- **真机测试脚本命名规范**：`xxx_test.py`，xxx 与对应设备模块名一致（如 `servo.py` → `servo_test.py`）。测试脚本不导出任何类，仅作为独立命令行脚本运行，不在 `__init__.py` 中引入。
- **`cam` vs `camera`**：`main.py` 中 `camera` 是云台（`CameraMount`），`cam` 是拍照采集器（`Camera`），注意区分，不要混用。
- **`capture()` 是同步阻塞**：在 FastAPI 异步路由或 `_handle_action` 中必须用 `asyncio.to_thread(cam.capture)` 包裹，否则会阻塞事件循环。
